# PostgreSQL 演示数据库：航空运输

## 简介

我们为您呈现一个 PostgreSQL 的演示数据库。我们以**航空运输**为研究对象。

该数据库可以用于：

* 自学 SQL 查询语言
* 编写关于 SQL 语言的书籍、手册和培训课程
* 在文章和笔记中展示 PostgreSQL 的功能

在开发演示数据库时，我们追求了几个目标：

* 数据模式应足够简单，无需太多解释即可理解。
* 数据模式必须足够复杂，以支持有意义的查询。
* 数据库中应填充类似真实数据的数据，增加趣味性。

演示数据库以 **MIT 许可证**发布。欢迎通过 edu@postgrespro.ru 向我们发送你的意见和建议。

## 安装 (2025年9月1日版本)

演示数据库有四个版本，数据量有所不同：

* `demo-20250901-3m.sql.gz`（133 MB）—— 三个月的飞行数据（数据库大小约 1.3 GB）
* `demo-20250901-6m.sql.gz`（276 MB）—— 六个月飞行数据（数据库约 2.7 GB）
* `demo-20250901-1y.sql.gz`（558 MB）—— 年度航班数据（数据库大小约 5.4 GB）
* `demo-20250901-2y.sql.gz`（1137 MB）—— 两年飞行数据（数据库容量约 11 GB）

小型数据库适合写查询且不占用大量磁盘空间。大型数据库能让你感受查询在大量数据上的表现，并思考优化。

文件包含一个 SQL 脚本，用于创建数据库并填充数据（实际上，它是工具创建的备份）。数据库将归执行脚本的 DBMS 用户所有。例如，要创建一个小型数据库，可以在程序中以 `postgres` 用户身份运行脚本：

```bash
gunzip -c demo-20250901-3m.sql.gz | psql -U postgres

```

> **注意：** 如果 `demo` 数据库已经存在，它将被删除并重新创建！最小支持的 PostgreSQL 版本是 15。你可以用生成器创建自己的演示基底版本。

## 与前一版本的区别

* **航线网络扩展**：已扩展至17个国家（俄罗斯、中国、印度、美国、加拿大、日本、法国、德国、意大利、英国、智利、瑞典、尼泊尔、芬兰、新西兰、奥地利和捷克共和国）。
* **机场数量增加**：表中机场数量已增加到约 5500 个。路由网络中机场不到一百个，但表格中行数众多，使你能够演示使用 GiST 和 SP-GiST 方法对点的索引。
* **新增国家字段**：`airports_data` 表格中新增 `country` 列（`jsonb` 格式），带有翻译，类似于 `airport_name` 和 `city`。演示数据库中没有单独的国家目录。
* **表名更改**：`aircrafts` 表重命名为 `airplanes`（因为英文中 aircraft 单复数同形，原名不准确）。
* **机队更新**：使用了更新、更大的飞机。十款模型中有两款是货机，不参与客运。
* **字段移除**：从 `tickets` 表中移除了 `contact_info`，因为其简单的结构无法展示 JSON 功能。
* **乘客标识**：`tickets.passenger_id` 唯一标识乘客（包含国家代码和数字），并与 `passenger_name` 对应。
* **登机与机票逻辑**：
* 仅对机票上的第一个航班办理登机，但会一次性发放所有航班的登机牌。
* 返程航班发放单独的机票。`tickets.outbound`（布尔值）标记直飞（`true`）和返程（`false`）。
* 增加了 `book_ref`, `passenger_id`, `outbound` 的唯一性约束（每位乘客只能预订一张去程和一张返程票）。


* **表名与字段名精确化**：`ticket_flights` 重命名为 `segments`，`amount` 重命名为 `price`。
* **转机限制**：最多支持四次转机。
* **时间维度与路由表**：`routes` 成为定义路线的独立表。路由随时间变化，因此主键是时间维度的（包含 `route_no` 和 `validity` 区间）。
* **表连接更新**：`departure_airport` 和 `arrival_airport` 移至 `routes` 表。`flights` 与 `routes` 的连接条件更新为：`routes.route_no = flights.route_no AND routes.validity @> flights.scheduled_departure`。
* **状态更新**：航班状态（`flights.status`）在登机时新增 `Boarding` 状态。
* **登机牌更新**：`boarding_passes` 表新增 `boarding_time` 字段。
* **视图更新**：新视图 `timetable` 隐藏了复杂的时间连接，并输出机场时区的时间。
* **数据类型调整**：多个字段统一使用 `text` 类型替代 `char`/`varchar`（如 `passenger_id`, `status`, `route_no`, `seat_no`, `fare_conditions`）。
* **错过转机**：起飞前不到 40 分钟抵达机场的乘客，不能搭乘同一机票的该航班或后续航班。

## 数据模式

* **预订 (`bookings`)**：主要实体。
* **机票 (`tickets`)**：同一预订可包含多名乘客，每人获得一张直达方向的机票，购买往返票则单独获得往返票。包含唯一票号和乘客信息（文档编号唯一标识乘客）。
* **航段 (`segments`)**：如果需要转机，机票包含多趟航班。
* **机场与航班 (`airports`, `flights`)**：航班从一个机场飞往另一个。同一航线号起降地相同但日期不同。
* **航线 (`routes`)**：航线网络每月更新。查询时需考虑航班时间和航线有效期的匹配（可使用 `timetable` 视图简化）。
* **登机牌 (`boarding_passes`)**：乘客办理首航登机后，获得所有航段的登机牌（标明座位和登机时间）。
* **座位与飞机 (`seats`, `airplanes`)**：座位数量和客舱布局取决于飞机型号。

## 数据表与视图详解

### 飞机预订展示 (`airplanes` 视图)

每种飞机型号均以三位数 IATA 代码标识。`model` 字段的值取决于所选语言（`bookings.lang`）。

| Столбец (列) | Тип (类型) | Описание (描述) |
| --- | --- | --- |
| airplane_code | char(3) | 飞机代码，IATA |
| model | text | 飞机型号 |
| range | integer | 最大飞行航程，公里 |
| speed | integer | 巡航速度，公里/小时 |

```sql
-- 视图定义：
SELECT airplane_code,
   model ->> lang() AS model,
   range,
   speed
FROM airplanes_data ml;

```

### 表 `bookings.airplanes_data`

包含飞机模型不同语言翻译的 `jsonb` 格式底表。

| Столбец (列) | Тип (类型) | Допустимость NULL | Описание (描述) |
| --- | --- | --- | --- |
| airplane_code | char(3) | not null | 飞机代码，IATA |
| model | jsonb | not null | 飞机型号 |
| range | integer | not null | 最大飞行航程，公里 |

* **主键**: `PRIMARY KEY, btree (airplane_code)`
* **检查约束**: `CHECK (range > 0)`, `CHECK (speed > 0)`

## 核心内置函数

### `bookings.now()`

返回演示数据所处的“当前时间”快照。以此基准点划分过去和未来的航班。

### `bookings.lang()`

返回当前系统设置的语言（用于翻译字段）。

```sql
-- 切换到俄语环境：
SET bookings.lang = 'ru';
-- 或者在数据库级别修改：
ALTER DATABASE demo SET bookings.lang = 'ru';

```

### `bookings.version()`

返回当前演示数据库的版本信息。

> 当前版本：`PostgresPro 2025-09-01`

**查看所有飞机型号：**

```sql
SELECT * FROM airplanes;

```

```text

---------------+------------------------+-------+-------

 339           | Аэробус A330-900neo    | 13300 |   870

 35X           | Аэробус A350F          |  8700 |   903

 77W           | Боинг 777-300ER        | 14600 |   905
 789           | Боинг 787-9 Dreamliner | 14000 |   913
```

**查看飞机座位布局（Embraer E170 商务舱）：**

```sql
SELECT *
FROM seats
WHERE airplane_code = 'E70' AND fare_conditions = 'Business';

```

**统计机场、城市和国家数量：**

```sql
SELECT
    count(*) airports,
    count(distinct country||','||city) cities,
    count(distinct country) countries
FROM airports;

```

```text
 airports | cities | countries
----------+--------+-----------
     5501 |   5157 |       230

```

**查询特定城市的机场（例如：莫斯科）：**

```sql
SELECT airport_code, airport_name, coordinates, timezone
FROM airports
WHERE country = 'Россия' AND city = 'Москва';

```

**获取当前数据库时间快照：**

```sql
SELECT bookings.now();

```

```text
          now
------------------------
 2025-12-01 00:00:00+03

```

**利用 `timetable` 视图查找未来最近的一班航班（SVX 到 WUH）：**

```sql
SELECT *
FROM timetable t
WHERE t.departure_airport = 'SVX'
  AND t.arrival_airport = 'WUH'
  AND t.scheduled_departure > bookings.now()
ORDER BY t.scheduled_departure
LIMIT 1 \gx

```

**查看具体预订详情与航段记录：**

```sql
SELECT r.route_no,
    dep.airport_code dep_airport, dep.country dep_country, dep.city dep_city,
    arr.airport_code arr_airport, arr.country arr_country, arr.city arr_city
FROM segments s
    JOIN flights f ON f.flight_id = s.flight_id
    JOIN routes r ON r.route_no = f.route_no AND r.validity @> f.scheduled_departure
    JOIN airports dep ON dep.airport_code = r.departure_airport
    JOIN airports arr ON arr.airport_code = r.arrival_airport
WHERE s.ticket_no = '0005433348362'
ORDER BY f.scheduled_departure;

```
